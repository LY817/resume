• 智行应用开发PaaS平台

    • 【项目介绍】
    为二航局企业应用全生命周期管理和监控的一站式PaaS平台，部署在自建的K8S集群上
    整合二航局企业应用的通用功能接口（如审批流、人员组织机构信息、物联网数据、合同等），作为微服务接口或成型的功能模块提供给企业应用使用
    支撑局内十余个项目应用（企业数字化和智慧工地）的开发和交付，提高开发的效率
    集成运维管理中间件，为开发人员提供统一的运维入口：DevOps应用发布流水线、kubernetes资源管理、日志监控等功能
    📆  项目周期：2020-09 ~ 2021-06
    ⚙️  技术选型
    - 业务开发组件：SpringCloud Alibaba
      网关：SpringCloud Gateway
      注册中心和配置管理：Nacos
      远程调用：open-feign
      限流控制：Sentinel
      分布式事务：Seata
    - 物联网平台：Jetlinks物联网平台的二次开发
      基于Project Reactor技术栈的非阻塞响应式调用
      http RESTful接口：Webflux、物联网网络组件：vert.x、数据库驱动：r2dbc
    - 运行环境 & DevOps
      Docker + kubernetes + Jenkins
    【职责描述】
    负责PaaS平台的物联网接入的能力建设：平台二次开发和设备协议对接
    参与老系统通用业务接口的微服务改造：业务逻辑梳理与实现
    参与基于PaaS平台开发的企业应用的需求分析、技术方案评审和技术支持
    【设计细节】
    💡 kube-apiserver接口整合
    当物联网微服务创建一个新的设备网关（使用vert.x监听一个新的端口）时，
    会调用封装kube-apiserver的客户端的微服务接口，对物联网微服务的service对应的configmap进行操作
    自动创建对应的tcp/udp端口转发关系（删除同理）

    【我的收获】
    - 基于Docker + kubernetes的云原生技术落地与实践经验
    - Java中基于Project React的响应式编程的实际使用经验
    - 在对开源项目（物联网平台）的消化和二次开发过程中，通过与社区大佬交流以及阅读代码，对设计模式和代码组织方式有更深的理解
    - 熟悉了Vue（PaaS平台使用的avue开发框架）和React ant Design（物联网项目前端）的业务开发


  • 二航局财务云业财一体集成对接

    • 【项目介绍】
    中交二航局财务系统升级对接，将二航局所有线上业务的单据接入到浪潮财务云系统，实现全局数十家子公司、数百项目部的财务数据的统一管控
    根据财务云对接过程中的新需求对业务系统进行开发和改造
    📆  项目周期：2021-06 ~ 2021-11
    ⚙️  技术选型
    - 使用RocketMQ作为业务系统和财务云系统业务单据对接的单据流转的中间件
    - 使用SpringBoot + Vue作为新业务系统/数据对接服务的开发框架
    - 使用h3c绿洲平台作为基础数据ETL处理的图形化工具
    【职责描述】
    负责与财务云、集团主数据以及其他平台进行接口对接和数据集成
    财务云业务对接过程中产生的新需求的需求分析和功能实现
    上线后运维和数据问题处理
    【设计细节】
    💡 设计模式使用
    来组织合同评审系统的28类合同的业务代码，解耦合、提高开发效率
    - 策略模式：定义通用的接口，来组织合同评审系统的28类合同的业务代码，作为component注入到Spring容器的一个map中
    - 模板模式：将通用合同的实现作为其他27类合同的基础实现，其他的合同在其基础上做扩展，提高代码的复用性和可维护性
    💡 单据流转控制
    - 使用RocketMQ同步刷盘、同步复制设置以及重试、死信队列机制，保证数据对接过程中的报账单据流转的可靠性
    【我的收获】
    - 在复杂的多方对接过程中，锻炼了自己的沟通表达能力
    - 锻炼了项目管理的相关技能：需求分析、风险评估等
