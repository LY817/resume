• 湖北电信聚翼智慧社区

    • 【概述】
    作为社区信息汇聚平台，将人、地、事、物等要素全部纳入网格进行管理，汇聚不同设备厂商的数据，发掘并发挥数据的价值，增强社区治安管理，提升物业服务质量
    【职责描述】
    我作为项目的主要开发者，全程参与了项目从0到1的整个过程：
    - 产品业务需求分析
    - 功能的设计与落地实现
    - 安防设备厂家的接口对接联调
    【项目介绍】
    📆  2018-11 ~ 2019-03 原型开发
    开发原型产品，并在几个湖北电信的园区中开始试点使用
    ⚙️  技术选型
    后端技术栈使用SpringBoot + Dubbo + MyBatis
    初期分为前端接口服务和数据库操作服务，服务之间通过Dubbo远程调用，通过multicast组网广播实现服务发现
    使用MySQL作为数据存储
    使用Redis保存用户登录信息以及字典缓存
    使用fastDFS存放上传的图片文件
    📆  2019-04 ~ 2020-06 功能迭代/重构改造
    逐步在武汉以及湖北地级市推广，接入的社区数量开始增加。随着系统访问量的上升，原来的"all-in-DB"的架构模式逐渐跟不上业务的性能需求，引入其他中间件和分布式解决方案来优化系统性能
    解耦/拆分:根据业务领域划分子模块
    将最初的一个数据库操作服务按照业务领域拆分为几个单独Dubbo服务
    将设备厂家的对接逻辑独立为可以单独部署的应用，通过事件总结与业务系统交互，使用微服务组件来统一管理各个服务的状态和配置
    ⚙️  技术选型
    - 使用RocketMQ作为业务事件总线
    - 使用Nacos来做配置管理和服务管理
    【细节/亮点】
    💡 事件总线
    - 数据分发：数据对接过程中，解耦了业务系统与多家设备厂家的对接逻辑；通过消息中间件实现"柔性远程调用"，提高数据分发过程的可靠性
       根据设备对接的业务需求，定义消息生产者和消息消费者之间交互的"API"
       根据tag来定义数据模型和增删改操作、根据topic来区分厂家
       消息生产者：在业务系统逻辑中使用自定义注解 + AOP实现在业务逻辑中埋点，将操作动作和操作参数作为消息发送到MQ
       消息消费者：利用SpringBoot的SPI机制实现自定义的starter
       根据消息生产者和消息消费者之间定义的交互"API"定义了一套对应的Java接口暴露给对接开发者，接口的实现类作为component注册到Spring容器中
       封装消息消费的路由逻辑，根据消息tag映射到对应接口方法，利用反射调用Spring容器中对应实现类的方法
       根据对应数据模型的主键作为路由key和客户端顺序消费API来保证数据操作的顺序性
       利用死信队列将重试失败的消息生成告警进入人工维护兜底
    - 事件驱动：实现事件相关业务场景如布控、告警，取代原来数据库轮训的实现方式
       将安防设备上报的人脸识别、车辆通行记录作为事件源，业务系统定义统一的事件上报接口，将上报数据投递到RocketMQ
       对接服务根据对接厂商不同的事件获取实现形式获取通行记录数据，经过数据映射、图片上传，调用统一的接口上报
       入库服务、布控规则服务、前端轮播WebSocket推送以及后面新增的下游业务都作为一个独立的消费者组消费通行记录topic
    💡 服务治理
    使用Nacos来管理厂家对接服务的配置
    安防厂家SaaS平台rest接口对接模式、设备直连模式、云平台对接
    【收获】
    - Dubbo、RocketMQ、MongoDB等分布式系统的相关解决方案的实践经验
    - 实践了SpringBoot使用的各种姿势


  • 湖北电信聚翼物联网管理平台

    • 【项目描述】
    接入中国电信物联网平台为基础实现对厂家、产品、终端、数据和物联网卡的管理
    【周期：2018-05 ~ 2018-10】
    【技术选型】
    - 基于Spring + SpringMVC搭建，Weblogic做web容器
    - 数据存储：Oracle + MongoDB
    - ORM框架：JPA + Spring jdbcTemplate/mongoTemplate
    - 数据访问使用Hibernate
    - RPC调用：web层与数据访问层内外网分离，之间使用Dubbo的通信协议替代http调用，服务发现使用组播模式
    - 消息中间件：使用RocketMQ 物联网设备上报数据流转
    【职责描述】
    》web功能模块开发
    - 物联网设备、产品、厂商、用户、套餐等数据模型的增删改查操作
    》接入物联网设备上报数据解析、存储以及展示
    - 利用MongoDB存储设备上报
    - 电动车GPS数据 利用2d索引实现位置消息查询
    【成果/收获】
    - Dubbo使用经验
    - 熟悉了RocketMQ、MongoDB等中间件
